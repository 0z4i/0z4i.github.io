telegram-message-to-id

The myth of Telegram's anonymity attracts users who believe that it depends solely on the platform, but being 100% anonymous is a pipe dream. Some Telegram users perform bad activities and try to cover their tracks by changing the profile information, like the username, name, or profile pictures, or by not setting a username and only using a name.

Name: can be anything and not unique is set in the account creation Username: It is optional to use and has the format @username; it is unique

But the account's foothold is traceable, no matter the username change or if they leave the communities (groups, channels) where they send the messages.

Also in some cases it's possible to extract the message history even if the account is deleted.

In this post I'll describe how to get the identity of a Telegram user using iterative cycles of data collection, creating new inputs with the analysis of the previously collected information.

Based on research into a previously identified community (group), I identify a user sharing content that is harmful to a vulnerable group; here the sender (user) has a username set, which makes the identification of the accounts easier.

The next step is to get the account details; this is possible with or without the username. For users without a username, open a conversation with the user in Telegram Web, and the URL will show the ID after a #.

But in this case I use the username, with a BOT, the funstat telelog. This tool mixes the Telegram API capabilities with scrapping and a DB of historical data like users' information and messages, then with the user I got: - Telegram account ID - Usernames History (3 usernames) - Names History (2 names) - Account creation date

Also, I check the Telegram profile, but the profile picture is created with AI (maybe not useful), and I check for other pictures, but none are obtained.

With the account information, I use the ID to trace the account and check for possible leaks about Telegram accounts, names, and usernames for possible reuse. The ID is very important because, unlike the username and name, the ID never changes, and we can find the account after changes.

Now I do a pivot over the Telegram information; with the ID, it is possible to get the phone number linked to the Telegram account (leak). For this account, it was possible, and with the extracted phone number, I got the country code (MX), city code, and the carrier.

With this I got (possibly not confirmed): - User country - User City

Using other published information (leaks), I search the phone number in these sources and return an account in the platform VK with a partial name, where the used last name matches one of the usernames in the Telegram username history, for example: - Telegram username: @vazquez_sr - VK name: Adrian Vazquez

The obtained information is expanded now with a partial name (one name and one last name); it makes it possible to create new terms to search with a possible city to filter results.

Using the past username (matched with the VK name) as a search term for possible username reuse, I found two new accounts, TikTok and Instagram, and the TikTok profile picture matches with one posted on the Instagram account. The name (name and last name) in the bio/description matches with the VK name.

Also on the Instagram highlights there is a video whose visual elements allow the use of a reverse image search for an IMGINT process to geolocate where the video was recorded, and the found place matches the city code found in the Telegram phone number.

At this point, I got more confirmations for: - The possible name - User location(approximate area) - The user face(based on posted images)

With the possible name and location, I searched directly on Facebook, and I got a profile that matches the name, and the profile picture is a picture also posted on the Instagram account. In this case, the name used in Facebook has another last name between the previously found ones, for example: - VK name: Adrian Vazquez - Facebook name: Adrian Schuroff Vazquez

The hometown mentioned on Facebook matches the phone's city code and the geolocated video, and it also mentions another city from a different state as the current location.

With all the collected information at this point of the investigation, it's possible to create new tuned inputs for a new iteration and perform activities, like searching for the potential parents of the target or searching for the target in published databases, but here it's needed to analyze all the collected data to discard unuseful information and structure it to create new terms of search because I have the following matches:

All of these connections allow us to expand the links between the found entities and perform a new collection cycle or end the collection phase to pass an analytic phase.
